# 邀评发送日志（SEND LOG）

用途：记录“发送-跟进-回复-处理”的全过程，便于后续写论文附录与致谢。

## 填写规则

- 每次发送或跟进都新增一行，不覆盖历史。
- `问题ID` 对应 `TARGET_EXPERTS.md` 中该专家的“唯一问题”。
- `处理状态` 只用固定枚举，避免后期统计混乱。

## 字段说明

- 日期：`YYYY-MM-DD`
- 动作：`首发` / `跟进` / `回复记录`
- 专家ID：如 `A01`、`B03`、`C05`
- 专家姓名：可匿名写缩写
- 问题ID：如 `Q-A01-1`
- 渠道：`Email` / `Scholar` / `ResearchGate` / `Other`
- 结果：`已发送` / `未送达` / `已回复`
- 处理状态：`待处理` / `已采纳` / `部分采纳` / `不采纳`
- 备注：最多 1 句，写最关键事实
