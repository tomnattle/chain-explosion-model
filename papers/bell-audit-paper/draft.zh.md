# 标题（中文）

基于分母审计的 Bell/CHSH 统计稳健性分析：严格协议与标准协议的对账研究

## 摘要

本文提出一套面向 Bell/CHSH 统计的分母审计（denominator audit）框架，目标是在同一数据源下识别“机制变化”与“协议簿记变化”对结论的相对贡献。方法上，我们并列计算严格协议（strict protocol）与标准协议（standard protocol），并将判定分为工程门槛（engineering gate）与论文门槛（thesis gate）两层，以避免将可计算性直接等同于可发表结论。在同一份 NIST completeblind 事件流上，二值 CHSH 得到 `S_strict=2.336276`（窗口 `0.0`）与 `S_standard=2.839387`（窗口 `15.0`），仅配对规则变化就导致 `Delta=+0.503111`。Bootstrap（`n=2000`）给出 `CI95_strict=[2.295151, 2.378669]` 与 `CI95_standard=[2.820420, 2.857413]`；由于标准窗口区间包含 `2sqrt(2)=2.828427`，本文不主张 Tsirelson 违背。结合模拟回退配置（`S_standard` 未高于 `S_strict`），结果表明 CHSH 指标对窗口和样本包含规则高度敏感。本文据此给出方法级结论：高 `S` 值本身不足以支撑本体外推，需与协议定义、不确定性区间和负结果共同审阅。

## 1. 引言

Bell/CHSH 结果常被用作讨论局域实在论边界的重要证据，但在实际数据处理中，时间窗、配对规则、归一化分母与后选择机制会共同影响最终统计量。若只聚焦单一 `S` 值而忽略协议簿记，容易把“口径变化”误读为“机制突破”。

本文的目标是把 Bell 结果放回可审计账本：把 strict 与 standard 两种协议并列计算，分离工程可计算性与论文论证门槛，并给出当前证据可支持与不可支持的范围。本文贡献在于提供可追溯的协议对账框架，而不是提出新的本体结论。

## 2. 方法

### 2.1 指标定义注册

本文采用 CHSH 标准形式 `S = E(a,b)+E(a,b')+E(a',b)-E(a',b')`，同时注册两种协议口径：

1. strict：更严格配对窗口；
2. standard：较宽的标准窗口。

二者共享同一数据源，但样本包含规则不同。本文将其视为“协议差分审计”，而非互相替代。若某协议出现高 `S`，仍需检查是否满足预注册门槛（例如 strict 上限约束与双协议一致性要求）。

### 2.2 实验协议

当前纳入两组代表性结果：

- NIST 索引配置（真实 CSV 源）：
  - strict：`pair_count=136632`，`S=2.336276`
  - standard：`pair_count=148670`，`S=2.839387`
  - strict bootstrap CI95（`n=2000`）：`[2.295151, 2.378669]`
  - standard bootstrap CI95（`n=2000`）：`[2.820420, 2.857413]`
  - 配对规则差值：`S_standard - S_strict = +0.503111`
- 模拟回退配置（simulated fallback）：
  - strict：`S=2.017316`
  - standard：`S=2.008601`

门槛采用双层：

- engineering gate：可计算性与样本量门槛；
- thesis gate：严格协议上限与双协议关系约束。

该分层用于避免把“可运行结果”直接提升为“论文结论”。

## 3. 结果

### 3.1 主结果

在 NIST 索引配置中，工程门槛通过但论文门槛未通过，主因是 strict 协议下 `S=2.336276` 超出预设上限约束（`strict_max_S=2.02`）。该现象说明：

1. 当前结果可以支持“协议下可计算并出现高 S”；
2. 但不足以支持“按预注册论文判据完成闭环”。
3. 在同一事件流上仅改变配对窗口就出现 `+0.503111` 差值，提示系统性分析规则敏感性。

### 3.2 稳健性

模拟回退配置显示 `S_standard` 未显著高于 `S_strict`，进一步提示“窗口与协议口径”对结论方向有实质影响。该结果强化了负结果保留原则：不应只保留“更高 S”的单一口径。
同时，真实数据 bootstrap 区间显示标准窗口 `S` 虽高，但区间仍覆盖 `2sqrt(2)`，因此不应表述为 Tsirelson 违背。

## 4. 讨论

本文结果强调了三个层次：

1. 数值层：存在高 `S`；
2. 协议层：高 `S` 发生在特定口径下；
3. 论证层：是否满足预注册论文门槛是独立问题。

从审计角度看，负结果（thesis gate 未通过）不是失败，而是保证研究可信度的核心证据。它防止将“统计现象”直接外推为“本体结论”。

## 5. 结论

可确认结论：

- Bell/CHSH 结果对协议定义高度敏感；
- 当前配置下工程可计算性成立，但论文门槛未闭环；
- 需要显式对账 strict/standard 及其门槛约束。

不可外推结论：

- 不能仅凭单次高 `S` 宣称机制突破；
- 不能把工程通过直接等同于论文级证明通过。

边界声明（显式）：

- 本文证明的是 Bell/CHSH 读数对分析规则（尤其配对窗口与统计定义）的敏感性。
- 本文结果本身不足以单独推出“替代量子理论本体”结论。
