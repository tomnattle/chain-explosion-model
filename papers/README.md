# Papers Workspace

这个目录用于并行准备两篇论文：

- `bell-audit-paper`: Bell/CHSH 审计与指标对账
- `ghz-threebody-paper`: GHZ 三体审计与分母机制分析

## 共同最小标准（投稿前）

1. 可复现：固定 seed、固定命令、固定依赖版本。
2. 可审计：原始结果 JSON + 汇总表 + 图表来源脚本一一对应。
3. 可解释：清晰区分“指标定义变化”与“物理命题变化”。
4. 可反驳：提供负结果与失败配置，不只保留最好结果。

## 建议写作节奏

1. 先写方法与审计协议（最稳定）。
2. 再写结果与对照（A/B 与多 seed）。
3. 最后写讨论与边界声明（避免过度结论）。
