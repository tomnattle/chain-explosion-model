## 一、工作定义（本项目是什么）

- 用可复现脚本与固定输出，审计 **Bell / CHSH、GHZ（Mermin 等）、公开实验数据重算** 中的**统计对象与测量实现**是否一致。
- 不把“程序跑出某数”等同于“裁决基础物理”；结论必须带**指标定义、样本保留、口径对照**。
- 自我定位：**方法贡献与测量协议敏感性**，而非替代标准量子形式体系的宣言。

## 二、认识论与立场（压缩表述）

- **盲人摸象**：在证据与语言均有限的前提下，任何表述（含教科书应用）都有边界；讨论焦点是**在给定定义与数据下，链条是否自洽、可复现**。
- **审其用、不审其神**：定理与形式体系作工具与对照尺；审计针对的是**测量、门控、归一化、样本选择**等环节，而非全盘否定推导结构。
- **直觉与公式**：允许直觉先行提出假设；必须用**可复现结果、负例、多尺子**约束；与结论直接相关的定义仍需补到可辩护程度。
- **局域性直觉**：动机上倾向不接受“超距作用”作为未加说明的默认；技术路径上体现为**局域传播模型 + 对统计口径的审计**，不以口号替代推导。

## 八、Δ映射物理闭合验证（论文A优先）

本节给出一套可直接写入论文A的方法学模板：目标不是先证明某统计量“唯一正确”，而是把 `Δ` 映射从工程假设提升为可操作、可复现、可证伪的实验定义。

### 8.1 目标与主张边界

- 目标：为 `Δ` 提供明确实验操作定义，并验证其在给定测量流程下是否闭合（定义一致、量纲一致、误差可追踪）。
- 允许主张：在该定义下，比较 `binary / continuous_raw / NCC` 三种统计量对 `S` 的影响。
- 禁止主张：在未完成闭合验证前，不声称 `NCC`（或任一口径）为“物理上唯一正确”。

### 8.2 操作定义（需要写进正文）

- 输入对象：逐次事件记录（时间戳、设置、原始响应、筛选标志），并固定预处理规则版本。
- `Δ` 定义：给出从原始事件到 `Δ` 的逐步映射公式，包含单位、归一化因子、截断/饱和规则。
- 观测量对应：说明 `NCC` 分母（如 `<A^2>`）在实验上如何由同批样本估计，及其不确定度估计方法。
- 误差传播：写明 `Δ -> E -> S` 的误差传递链（解析式或 bootstrap 均可，但需固定方案）。

### 8.3 物理闭合判据（最小可发表集）

通过以下四项可判定“闭合成立”：

- 定义闭合：同一输入在不同运行环境得到一致 `Δ`（数值差异在容忍阈值内）。
- 量纲闭合：所有中间量单位自洽，归一化后量为无量纲，且无隐式尺度漂移。
- 流程闭合：不同合法预处理顺序（等价变换）下，`S` 结论保持稳定。
- 统计闭合：bootstrap / seed sweep 下，三口径差异方向稳定，不由单次采样偶然触发。

### 8.4 失败条件（可证伪性，必须前置写明）

任一条件触发即判定该版 `Δ` 映射失败：

- 小幅预处理改动导致 `S` 结论翻转（例如 violation 与 non-violation 互换）。
- `NCC` 分母估计对样本筛选高度敏感，且无可辩护物理理由。
- `Δ` 对时间窗或配对策略出现异常脆弱性（参数微调即大幅漂移）。
- 不同实现路径无法复现实验结果（超出预设数值容忍阈值）。

### 8.5 对NIST段落的降格规范（当前建议）

- 在 `Δ` 闭合完成前，NIST结果标注为 `provisional`（初步证据）。
- 正文保留 RMSE 对比，但同步声明：该结果依赖当前 `Δ` 工程映射，尚非物理闭合证明。
- 结论段避免“支持/证伪某理论”的强措辞，仅陈述“统计口径敏感性已被观测”。

### 8.6 最小产出物（两周执行版）

- 文档：`Δ` 映射规范（公式、单位、流程、误差）。
- 代码：单命令复现实验脚本（固定配置+固定随机种）。
- 结果：主表（`S_binary`, `S_continuous_raw`, `S_continuous_ncc`）+ 稳健性附表。
- 报告：失败案例日志（至少包含一类触发失败条件的反例）。
