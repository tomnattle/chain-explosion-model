# 2D 映射策略规范（MAPPING_POLICY_2D）

本规范用于在进入 3D 扩展前，先锁定 2D（仅 `Δ` 轴）评估口径，降低结论漂移风险。

## 1) 目标

- 固定一个“主映射”用于对外主结论。
- 固定若干“压力映射”用于内部稳健性压力测试。
- 用统一门槛判断模型是否达到“可推进到 3D”的条件。

## 2) 映射定义

- 主映射（Primary）: `half_split`
  - 定义: `slot 0..7 -> +1`, `slot 8..15 -> -1`
  - 角色: 对外主口径与主图文叙述。
- 压力映射（Stress）:
  - `parity`
  - `quadrant_split`
  - 角色: 验证结论是否仅由映射约定驱动。

## 3) 评估口径（禁止作弊）

- 仅使用 `Δ` 轴聚合统计（2D）。
- 使用 `LOBO-CV`（留一bin外推）。
- 采用二项分布负对数似然（neg-loglik）作为泛化指标。
- 每次测试bin严格不参与拟合（no leakage）。
- 采用 bootstrap 估计赢家概率（`P(model wins)`）。

## 4) 通过门槛（Gate）

模型要进入 3D 阶段，需满足：

1. 主映射门槛  
   - 在 `half_split` 下 `P(model wins) >= 0.90`。
2. 压力映射门槛  
   - 在每个压力映射下，不允许出现灾难性退化（见第5条）。
3. 解释一致性门槛  
   - 主结论必须写成条件句：`在主映射定义下...`。

## 5) 灾难性退化判定

对任一压力映射，若模型同时满足：

- `P(model wins) <= 0.05`
- 且点估计 `CV neg-loglik` 明显劣于当期最佳模型（差值超过 2 倍）

则记为灾难性退化，Gate 判定为 `FAIL`。

## 6) 自动检查输出约定

- 自动检查脚本输出文件：
  - `artifacts/public_validation_pack/NIST_MAPPING_POLICY_CHECK.md`
- 必须包含：
  - 当前主映射赢家概率
  - 压力映射退化判定
  - Gate 总结：`PASS` / `FAIL`
  - 一句话决策建议：`继续2D加固` 或 `允许进入3D`

## 7) 当前项目状态（截至本规范落地）

- 主映射 `half_split`：`HighCos` 胜率高。
- 压力映射 `parity/quadrant_split`：赢家切换为 `Bell`。
- 结论：映射敏感性仍高，当前 Gate 状态应视为 `FAIL`，建议继续 2D 加固。
