# 叙事分层与章首体例修订方案（Round 3）

**日期**：2026-05-05  
**依据**：作者与 Claude 的对话纪要——章首「给普通读者」框打断叙事流；根因是正文跳跃而非缺框；正确做法是把信息**融进正文**；全书采用**分层阅读**而不拆成两本书。

---

## 一、问题诊断（对话共识）

1. **章首引用块过重**：像免责声明/教学大纲，破坏「跟着一个人走」的直觉流。  
2. **补丁思维**：因正文过跳才加框；长期应降低对框的依赖。  
3. **多身份稀释**：数据审计员、模型建造者、哲学口吻若并列堆砌，每边都弱。  
4. **价值分层仍要一本厚书**：不采用「只出薄本」切割，而用**路标**让不同读者自行决定读深读浅。

---

## 二、全书原则（给后续改稿与 Cursor）

| 原则 | 执行要点 |
| :--- | :--- |
| **叙事优先** | 每一章尽量是：普通人 → 做了一件事 → 发现一个问题 → 去追它。**避免**「本章将介绍…」的讲义腔。 |
| **术语就地一句** | Bell、NIST、CHSH、纠缠、符合窗、二值化、保留率等专业词：**首次出现或关键处**用正文里**一句话**说清，不单列释义清单。 |
| **章内不重复序言** | 「教科书常识」「主流共识」「本书不宣称推翻…」等**防御性套话**：序言已写则**章内不再复读**（边界在技术句里点到即可）。 |
| **章首无框** | 删除所有 **`【给普通读者 · 这幅图在说什么】`** 引用块；必要信息并入第一节或第一次需要处。 |
| **章尾无「小结」课** | 删除或并入上一节：**正文说完就收**；若必须承上启下，用**一两句**接在末段，不单立「小结」节。 |
| **引用框里的「挑剔的读者」** | 改为**一句正文**陈述边界，不单独成框（除非仓库别处体例强制保留「本章边界」短框——已比大导读轻）。 |
| **分层路标** | 在**第2、5、14章**正文收束处各加**简短路标**（见下），告知：故事线可停、数据/模型/展望各在何处继续。 |

## 四、分层路标文案（插入位置）

- **第2章末**（`03-chapter.md`）：若只关心「泡泡动机 + 为何盯公开数据与读数规则」，读完本章可**暂歇**；要看格子上的损耗地面，继续第3章；要直接跳到 **NIST / CHSH 审计主线**，可从目录进**第8章**起读。  
- **第5章末**（`09-chapter05.md`）：栅格玩具段告一段落；**主审计线**在 **第8–11章** 与公开数据、二值化、GHZ 协议；若你主要为 Bell/读数规则而来，**允许跳读**至第8章。  
- **第14章末**（`18-chapter14.md`）：**模型与探索脚本地图**至此；后续第15章为规划占位、第16章为失败记录、第17–18章为设想——若你**只要审计与诚实边界**，可侧重第8–10章与第16章，不必读完第三部分。

（路标为**建议稿**，成书时可再压字数。）
