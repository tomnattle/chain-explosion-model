## 2. Bell 不等式时间窗口审计

> * **研究目标**：量化审计时间窗口（Coincidence Window）、事件配对规则、筛选阈值对 CHSH 关联统计结果的影响，系统性验证实验结论对**观测协议（Observation Protocol）**的依赖程度。
> * **软件实现**：独立开发多维窗口扫描引擎、统计量横向比对模块及标准化审计报告导出组件；通过固定运算算子与统计口径，确保跨样本复现的**逻辑一致性**。
> * **独创性桥接公式（核心审计节点）**：$$E(a,b) = \frac{\langle \cos(\lambda-a)\cos(\lambda-b) \rangle}{\langle \cos^2(\lambda) \rangle}$$该公式系作者针对“涟漪连续投影解释”与“离散统计映射”开发的**特征转写算子**。本软件将其设为 Bell 节点的**强制核验锚点**，用于判定底层物理猜想与顶层实验统计的匹配度。
> * **关联算子本质差异声明（AI辅助反推）**：本系统并非经典偏振实验静态重放，而是“动态逻辑审计引擎”。关键差异在于：① 参数血统不同——传统写法中的 `λ` 通常视为外部随机输入，本项目将 `λ` 定义为由涟漪三元组演化得到的位移矢量；② 分母功能不同——分母不再仅作常数归一项，而被映射为涟漪阻抗模量 `𝒵_R` 的审计接口，用于追踪计算能耗与信息损耗平衡；③ 结果目的不同——目标不是单次曲线拟合，而是借由关联算子反向锁定可稳定收敛的参数口径。
> * **审计执行口径**：基于窗口动态扫描，提取单边/符合计数（singles/coincidences）并代入上述公式重构 $E_{\text{NCC}}$ 关联曲线。系统必须触发 **“映射自洽性校验”**：若实测统计偏离公式预测边界，自动触发 `expected-fail` 归档并标记逻辑待修正状态。
> * **可核验证据**：时间窗口扫描源码包、多参数批量实验 JSON 归档、Bell 协议敏感性分析报告。
> * **权利边界声明**：本项目主张保护上述自研审计流程、代码实现架构及报告组织表达；不主张对 Bell 不等式、CHSH 统计方法等公有领域数学理论的独占权。

- 软件实现：基于 V6 版本核心脚本，采用差分进化优化算法，自定义物理边界约束、惩罚项规则、收敛验收门槛；**三元组参数绝非人工手动填写，为算法全自动迭代求解生成**；额外新增全频域通透性审计模块，以标准玻璃透光窗口为参照，完成全域频段透过率对照分析。
- 可核验证据：V6联合约束求解源码、原始最优参数输出JSON、标准化精度摘要文档、全频域vs玻璃对照曲线、全域通透性审计报告。
- 核心固化判据：`triplet_distinct_from_glass_window` 介质差异化判定、`script_level_seated` 脚本级可复现锁定。
- 关键权利边界：**参数求解算法、约束规则、审计流程、文档编排、数据呈现形式受版权保护；μ/ρ/η 单纯数值本身为客观运算结果，不单独主张独占权利**。

- 研究目标：建立刚性审计规则，实现「合规条件稳定通过、异常条件必然失效」的双向验证，区分介质属性差异、排除同类带通机制混淆，提升成果可辨识性与抗质疑能力。
- 软件实现：集成多轮重启校验、变量遍历扫描、排列置换检验、自助统计分析、变量消融对照全套工具；细化三类预期失效场景：参数脱轨失效、协议不匹配失效、环境阈值过载失效；新增介质身份区分审计，量化输出窗内透过率、窗外透过率、全域最低透过下限，搭配expected-fail对照实验，杜绝单一指标误导。
- 阿喀琉斯之踵补强项（新增强制审计）：明确记录并单列验证等式  
  `E(a,b)=<cos(λ-a)cos(λ-b)>/<cos²(λ)>` 的实验可观测对应关系。  
  审计策略采用现有 Bell 原始数据中的 `coincidences` 与 `singles` 重分析，计算  
  `E_NCC = coincidences / sqrt(singles_A * singles_B)`，检验其随角差 `Δ` 是否呈余弦形状。
- 证据落地要求（补强门槛）：必须提供“连续投影假设 -> 点击率统计量”映射说明、`E_NCC(Δ)` 曲线图、拟合优度与失败样例；若不满足余弦闭合，需在报告中触发“模型待修正”标记，不允许静默通过。
- 可核验证据：可辨识性审计完整源码、高强度扩展组/极限条件审计结果、全局锚点结论文档、介质差异化量化数据表。
- 对应脚本入口（仓库现有）：`scripts/explore/explore_ncc_singles_coincidence_bridge.py`（用于 singles/coincidences 桥接重算）。
- 核心原创亮点：整套expected-fail双向失效验证机制，为本软件独有原创设计与核心特色。
