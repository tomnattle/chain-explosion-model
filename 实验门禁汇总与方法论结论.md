## 方法论结论（拟合 vs 共享参数 vs「作弊」）

1. **两条主线不宜混为一谈**  
   - **格点 CE 线**（`ce_*` 及大量 `verify_*` / `explore_*`）：在**固定局域传播规则**下看现象是否涌现，**通常不是**对某条外部实验曲线做逐点最小二乘拟合。  
   - **涟漪 / 三元组线**（`ripple_quantum_tests*`、`triplet_*`）：对 QM-like 参考曲线或光学数据做 **NRMSE / 形状 / 联合门禁** 与优化，文档明确 **toy bridge**、**\(\mu,\rho,\eta\) 在 SI 映射前为无量纲旋钮** 等——**方法论上更接近「在约束下重现数值/形状」**。

2. **「不是每个实验单独改模型」——仓库内部分设计在降低这种观感**  
   - 涟漪 **v6 联合**：**同一组** \((\mu,\rho,\eta)\)（及线宽等）驱动四条子实验的 toy 映射，用 **`joint_pass`** 等联合判据。  
   - **v2/v3**：形状与常数分层、`derived` 与 `calibrated` 对照。  
   - 部分审计脚本含**预注册 / 反事实应失败**等检查。  
   这些**不能数学上排除**所有隐蔽自由度；bridge **非第一性原理**，不同 `artifacts` 子目录仍可能对应**不同配置**。

3. **精确一句话**  
   - **是**：许多对外展示「与参考一致」的管线，**就是拟合 + 约束优化 + 玩具映射**，目标是 **reproduce**，读档时应按专论/README 的**解释边界**收束强度。  
   - **不全是**：**统一 CE 现象学**更强调**同一机制**能否产生同类行为，而非每个实验换一套完全不同的方程。  
   - **是否算作弊**：关键在是否**隐瞒**独立调参或**事后改门禁**；本仓库多处**明文写出** toy 层与旋钮边界，属于把**可证伪层级**写清楚，而非把 toy 匹配等同于**标准量子理论已被替代**。
